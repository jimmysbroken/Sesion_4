## Flujo de trabajo
** Flujo de trabajo elegido: Git Flow**
- ¿Por qué elegimos este flujo? 
Básicamente porque es más simple y directo. 
Main es nuestra rama principal y protegida, trabajmos en una rama y hacemos commits
y pushes. Se abre un PR para revisión y después de ella se hace un merge.
Consideramos un flujo de trabajo más lineal y rápido
